PREFLOP GAME multiway · equity-model postflop

Solves the preflop tree exactly — limps, any sizes, any player count — with postflop play priced by an equity-realization model (all-in terminals are model-exact). Multiway output is an equilibrium of the model, not gospel; the BR gap below is the convergence metric. Any line that reaches the flop heads-up can be sent straight to the postflop solver for an exact solve — ranges, pot and stack carry over.

1 · SET UP THE SCENARIO
2 · BUILD THE GAME
3 · SOLVE
4 · MODEL THE PLAYERS · optional

Give seats real-player tendencies: pick an archetype or enter HUD stats — ranges are generated by distorting this solve's own equilibrium — then tweak on the grid and APPLY. RE-SOLVE makes the table adjust around them; setting a HERO freezes everyone else so your seat's re-solve becomes a maximum-exploitation strategy. Profile solves run on the CPU engine. Workflow: solve the baseline once, pick archetypes for every seat you want modeled (generation distorts the CURRENT solve, so set them all up before applying), then just SOLVE — pending model changes apply automatically, and a changed table restarts convergence fresh so the grids reflect the new opponents.

5 · SET UP THE PREFLOP ACTION

Once solved, construct the hand in the panel to the right: each segment of the action ribbon lists every available action with its solved frequency — click one to advance the line, or click any earlier action to branch. When the line reaches the flop heads-up, step 6 unlocks.
